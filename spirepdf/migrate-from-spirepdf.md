## Spire.PDFから移行する理由

### 重大な技術的問題

Spire.PDFには、実際の使用に影響を与えるいくつかの根本的な問題があります：

| 問題 | 影響 | IronPDFの解決策 |
|-------|--------|------------------|
| **テキストが画像としてレンダリングされる** | PDFが検索不可、アクセス不可、テキストをコピーできない | 実際のテキストレンダリング |
| **Internet Explorerの依存** | 時代遅れのレンダリング、セキュリティリスク | 現代のChromiumエンジン |
| **フォント埋め込みの失敗** | 他のシステムで文書が正しく表示されない | 信頼性のあるフォント処理 |
| **大きな展開フットプリント** | 高いメモリ使用量、遅い起動 | 効率的な展開 |
| **限定的なCSSサポート** | 現代的なレイアウトが正しくレンダリングされない | 完全なCSS3サポート |

### 核心問題：画像ベースのPDF

Spire.PDFの`LoadFromHTML()`メソッドを使用すると、テキストが実際のテキストではなくビットマップ画像として**レンダリングされる**ことがよくあります。これは深刻な問題を引き起こします：

```csharp
// ❌ Spire.PDF - 画像ベースのPDFを作成
PdfDocument pdf = new PdfDocument();
pdf.LoadFromHTML("<h1>重要な契約</h1>", false, true, true);
pdf.SaveToFile("contract.pdf");

// 結果のPDFでの問題：
// - テキストを選択できません
// - テキストを検索できません
// - テキストをコピーできません
// - スクリーンリーダーで読み取れません（アクセシビリティ違反）
// - ファイルサイズがはるかに大きい
// - 拡大するとピクセル化します
```

### 機能比較概要

| 機能 | Spire.PDF | IronPDF |
|---------|-----------|---------|
| **HTMLレンダリング** | IE/Edgeベース（時代遅れ） | Chromium（現代的） |
| **テキスト出力** | 画像（選択不可） | 実際のテキスト（選択可能） |
| **CSS3サポート** | 限定的 | 完全 |
| **Flexbox/Grid** | サポートされていない | 完全サポート |
| **JavaScript** | 限定的 | 完全なES6+ |
| **フォント埋め込み** | 問題あり | 信頼性あり |
| **PDFアクセシビリティ** | 悪い（画像ベース） | 優れている |
| **現代の.NET** | .NET 6+部分的 | 完全な.NET 6-9 |
| **APIデザイン** | 複雑 | シンプルで直感的 |

## テキストが画像として扱われる問題

### これが重要な理由

Spire.PDFが画像ベースのレンダリングを使用してHTMLをPDFに変換すると、文書は重要な機能を失います：

**1. テキスト検索不可**
```csharp
// Spire.PDF LoadFromHTMLによって生成されたPDF
// ユーザーはCtrl+Fを使用してテキストを見つけることができません
// 文書管理システムはコンテンツをインデックスできません
// 法的発見ツールは文書を検索できません
```

**2. テキスト選択/コピー不可**
```csharp
// ユーザーが引用、参照、またはデータをコピーしようとする：
// 1. テキストを選択 → 何も起こらない（画像です）
// 2. 右クリック → 「テキストをコピー」オプションなし
// 3. 手動で全てを再入力する必要があります
```

**3. アクセシビリティ違反**
```csharp
// 画像ベースのPDFは失敗します：
// - WCAG 2.1準拠
// - セクション508準拠（米国政府）
// - ADA要件
// - スクリーンリーダーの互換性
// あなたの組織は法的責任を負うかもしれません！
```

**4. 大きなファイルサイズ**
```csharp
// 同じ内容の比較：
// Spire.PDF（画像ベース）：2.5 MB
// IronPDF（テキストベース）：150 KB
// それは16倍大きいです！
```

### 検出：あなたのPDFは画像ベースですか？

```csharp
// Spire.PDFで生成された文書を開きます
// これらのテストを試してください：

// テスト1：テキスト選択
// テキスト上でクリックしてドラッグします
// 何もハイライトされない場合 → 画像ベース

// テスト2：Ctrl+F検索
// ページ上の任意の単語を検索します
// 「一致するものが見つかりません」の場合 → 画像ベース

// テスト3：コピー/貼り付け
// テキストを選択してメモ帳にコピーします
// 何も貼り付けられない場合 → 画像ベース
```

### IronPDFの解決策

```csharp
using IronPdf;

// ✅ IronPDF - 実際のテキストPDFを作成
var renderer = new ChromePdfRenderer();
var pdf = renderer.RenderHtmlAsPdf("<h1>重要な契約</h1>");
pdf.SaveAs("contract.pdf");

// 結果：
// ✅ テキストは完全に選択可能
// ✅ テキストはCtrl+Fで検索可能
// ✅ テキストをクリップボードにコピーできます
// ✅ スクリーンリーダーが完璧に機能します
// ✅ ファイルサイズがコンパクトです
// ✅ 拡大してもクリアです
```
