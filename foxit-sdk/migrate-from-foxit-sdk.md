## IronPDFへの移行理由

### Foxit SDKの課題

Foxit PDF SDKは強力なエンタープライズレベルのライブラリですが、かなりの複雑さを伴います：

1. **複雑なライセンスシステム**：複数の製品、SKU、ライセンスタイプ（開発者ごと、サーバーごと、OEMなど）があり、適切なオプションを選択するのが難しい
2. **エンタープライズ価格設定**：大規模な組織向けに価格設定されており、小規模チームにとっては禁止的である
3. **手動インストール**：手動でのDLL参照またはプライベートNuGetフィードが必要であり、簡単な公開NuGetパッケージはない
4. **冗長なAPI**：ライブラリの初期化、エラーコードのチェック、明示的なリソースのクリーンアップがボイラープレートを追加する
5. **別のHTML変換アドオンが必要**：HTMLからPDFへの変換には追加のアドオンの購入が必要
6. **複雑な設定**：設定には詳細なオブジェクト設定（例：`HTML2PDFSettingData`）が必要
7. **C++の遺産**：APIパターンはC++の起源を反映しており、現代のC#では自然に感じられない

### IronPDFの利点

| 項目 | Foxit SDK | IronPDF |
|------|-----------|---------|
| インストール | 手動DLL/プライベートフィード | 単純なNuGetパッケージ |
| ライセンシング | 複雑、エンタープライズ向け | 透明性、全サイズ対応 |
| 初期化 | `Library.Initialize(sn, key)` | ライセンスキーを一度設定 |
| エラーハンドリング | ErrorCode enums | 標準の.NET例外 |
| HTMLからPDFへ | 別のアドオンが必要 | 組み込みのChromium |
| APIスタイル | C++の遺産、冗長 | 現代の.NETパターン |
| リソースクリーンアップ | 手動の`Close()`/`Release()` | IDisposable/自動 |
| ドキュメント | エンタープライズドキュメント | 公開チュートリアル |

### コストベネフィット分析

Foxit SDKからIronPDFへの移行は以下を提供します：
- **複雑さの削減**：よりシンプルなAPI、ボイラープレートの削減
- **開発の高速化**：直感的なメソッド、設定の削減
- **現代の.NET**：async/await、LINQの互換性、標準パターン
- **HTMLファーストアプローチ**：WebスキルをPDF生成に活用
- **包括的な機能**：HTML変換のための別のアドオンは不要

## 開始前に

### 前提条件

1. **.NET環境**：IronPDFは.NET Framework 4.6.2+、.NET Core 3.1+、.NET 5/6/7/8/9+をサポート
2. **NuGetアクセス**：NuGetからパッケージをインストールできることを確認
3. **ライセンスキー**：本番使用のためのIronPDFライセンスキーを取得

### プロジェクトのバックアップ

```bash
# バックアップブランチを作成
git checkout -b pre-ironpdf-migration
git add .
git commit -m "Foxit SDKからIronPDFへの移行前のバックアップ"
```

### Foxit SDKの使用箇所の特定

```bash
# Foxit SDKの参照をすべて見つける
grep -r "foxit\|PDFDoc\|PDFPage\|Library.Initialize\|Library.Release" --include="*.cs" --include="*.csproj" .

# Foxit DLLの参照を見つける
find . -name "*.csproj" | xargs grep -l "Foxit\|fsdk"
```

- ライセンスキーの場所と初期化コード
- カスタム設定と設定
- エラーハンドリングパターン
